# Head Tracked Fixed Object Demo

一个基于 `Three.js + MediaPipe` 的实时头部跟踪 3D 视窗 Demo。

这个项目模拟的是“透过一个窗口去看空间中的固定物体”：
- 中心方块固定在空间盒子中央，不会跟着头部漂移
- 头部上下左右移动时，视窗会像真实窗口一样改变观察视角
- 头部远近变化不会驱动方块缩放
- 单手/双手捏合手势可以继续旋转、扭转和缩放中心方块

## 主要功能

- 实时头部跟踪视角
- 固定物体透视观察
- 空间盒子网格纵深
- 单手捏合旋转
- 双手捏合缩放与扭转
- 手势状态可视化
- 全屏模式
- 参数面板调节
- 鼠标预览降级模式

## 运行环境

- Node.js `20.19.0` 或更高版本
- 建议使用最新版 `Chrome` 或 `Edge`
- 需要可用摄像头才能体验头部跟踪和手势识别

项目根目录包含 `.nvmrc`，如果你使用 `nvm`，可以直接切到推荐 Node 版本。

## Windows 一键运行

在 Windows 上，直接双击 `start-demo.bat` 即可。

前置条件：
- 已安装 `Node.js 20.19.0+`
- 浏览器可正常访问本地 `localhost / 127.0.0.1`
- 如果要体验头部跟踪和手势识别，需要可用摄像头并允许浏览器权限

一键启动的实际流程：
1. `start-demo.bat` 先切换到项目根目录。
2. 脚本检查本机是否存在 `node` 命令。
3. 然后调用 `node scripts/start-demo.mjs`。
4. `start-demo.mjs` 会检查当前 Node 版本是否满足要求。
5. 如果本地还没有可用依赖，会自动执行 `npm ci` 完成首次安装。
6. 如果本地依赖已经存在，则直接复用当前依赖，避免在 Windows 上因为文件占用导致重装失败。
7. 接着启动 `vite` 开发服务器。
8. 最后自动打开浏览器并进入本地页面。

如果你后续从 GitHub 拉到了新的依赖变更，而本地依赖还是旧的，脚本会优先复用现有依赖保证稳定启动；这时如果启动异常，手动执行一次 `npm install` 即可。

默认会打开本地地址：`http://127.0.0.1:5173`

相关文件：
- `start-demo.bat`
- `scripts/start-demo.mjs`

## 控制说明

### 头部透视

- `跟踪响应`
  控制视角跟随头部的速度。越高越灵敏，越低越平稳。

- `前馈补偿`
  根据最近一帧头部运动趋势轻微提前视角变化，用来减小延迟感。

- `头部位移幅度`
  放大或缩小头部上下左右移动对视角的影响范围。

- `透视强度`
  控制空间盒子和中心方块的视角变化有多明显。

- `空间盒子深度`
  调节视窗内部空间盒子的纵深长度。

- `视窗取景`
  控制窗口开口感和整体构图。

### 手势操控

- `手势操控`
  开启或关闭手势识别。

- 单手捏合
  旋转中心方块

- 双手捏合
  缩放中心方块

- 双手捏合并扭转
  扭转中心方块

### 按钮

- `进入全屏`
  让视窗铺满屏幕，全屏时隐藏控制元素

- `重新居中`
  以当前头部位置作为新的观察中心

- `重置物体`
  重置中心方块的旋转和缩放

## 常见问题

### 1. 页面提示“摄像头不可用”

请确认：
- 你是通过 `localhost` 或 `127.0.0.1` 打开的页面
- 浏览器已经允许摄像头权限
- 当前电脑上有可用摄像头设备

如果摄像头不可用，项目会自动降级到鼠标预览模式。

### 2. 页面打开但没有头部跟踪效果

请确认：
- 浏览器标签页处于前台
- 摄像头画面里能稳定看到人脸
- 光线足够，头部没有离开取景范围

### 3. 手势没有生效

请确认：
- 已开启“手势操控”
- 摄像头画面中能看到完整手部
- 拇指与食指有明确捏合动作
